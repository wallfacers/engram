# Feature Specification: 记忆引擎抽离(Memory Engine Extraction)

**Feature Branch**: `001-memory-engine-extraction`

**Created**: 2026-07-18

**Status**: Draft

**Input**: 把 workhorse-agent 的记忆引擎搬进 engram,成为自洽、可独立编译、可离线跑通的独立库,搬运保真、切断宿主耦合。纯搬运 + 行为保持,不重设计公开 API、不做算法改进。

## User Scenarios & Testing *(mandatory)*

### User Story 1 - 记忆引擎作为独立库可编译、可测试 (Priority: P1)

作为 engram 的维护者,我需要把 workhorse-agent 里已调优的记忆引擎作为一个**自洽的独立库**取出来,使它不再依赖任何宿主(会话、权限、工具生态)代码,能在 engram 仓库里独立构建、独立运行全部既有测试。这是所有后续集成面(MCP / CLI / SDK)的地基:没有这个可独立编译、行为可信的引擎,后面每一层都无从谈起。

**Why this priority**: 这是 engram 立项的第一块地基,也是 MVP。引擎不能独立站起来,项目就不存在。它独立可交付、可验证,且不依赖任何其它故事。

**Independent Test**: 在 engram 仓库内执行完整测试套件,全部通过;且引擎代码中不存在任何指向宿主(会话/权限/工具)包的引用。二者同时成立即证明本故事完成。

**Acceptance Scenarios**:

1. **Given** engram 仓库已包含抽离后的记忆引擎, **When** 在纯净环境(无宿主源码、无外网)执行构建, **Then** 构建成功产出可执行的引擎与评测工具,无缺失依赖、无对宿主代码的引用。
2. **Given** 抽离前记忆引擎在宿主仓库内自带的全部单元测试, **When** 这些测试随引擎一并搬入 engram 后执行, **Then** 全部通过,且测试内容与抽离前语义一致(仅调整了引用路径)。
3. **Given** 引擎依赖的一处宿主查询解析工具(原属宿主工具生态), **When** 检视抽离后的引擎, **Then** 该能力已内化进引擎自身,引擎不再引用宿主工具生态,且该能力行为与抽离前逐字节一致。

---

### User Story 2 - 检索行为在抽离前后逐条一致(保真证明) (Priority: P1)

作为对记忆质量负责的人,我需要**证明**这次搬运没有悄悄改变引擎行为——记忆质量是本项目唯一的差异化护城河,是用五轮消融换来的,任何"顺手改动"都可能无声抹掉调优成果。因此抽离后的引擎在相同输入下,必须返回与抽离前**逐条相同**的检索结果。

**Why this priority**: 与 US1 同为 P1。搬运若不保真,等于把护城河搬丢了;可信的保真证明是本次抽离能被接受的前提(对齐宪法「评测回归门禁」)。

**Independent Test**: 用固定语料、固定查询集、固定(或打桩)向量,让抽离后的引擎产出检索结果,与抽离前基线的排序结果逐条比对,完全一致即通过。该测试确定性、可重复、无需外部服务。

**Acceptance Scenarios**:

1. **Given** 一组固定的记忆语料、固定的查询集合、以及固定或打桩的向量表示, **When** 抽离后的引擎对每个查询执行检索, **Then** 返回的记忆条目标识按排序逐条比对,与抽离前基线完全一致。
2. **Given** 三路检索信号(语义 / 关键词 / 实体)中的任意一路缺失或失败, **When** 引擎执行检索, **Then** 其余信号独立优雅降级并仍返回结果,降级行为与抽离前一致(不整体报错)。
3. **Given** 上述保真对拍用例, **When** 纳入持续集成, **Then** 每次改动都会自动重跑该对拍,防止后续无声回退。

---

### User Story 3 - 评测/回归工具随引擎一并可用 (Priority: P2)

作为同时对产品回归和论文实验负责的人,我需要既有的长期对话记忆评测工具随引擎一起搬进 engram 并保持可用——它一份投入两份产出:既是引擎的回归测试,也是评测可靠性论文的实验基础设施。

**Why this priority**: P2。它不阻塞 US1/US2(二者靠确定性对拍与单测即可验收),但它是引擎长期质量守门与论文线的必要设施,应在本次抽离内一并到位而非另起炉灶。

**Independent Test**: 评测工具在 engram 内可构建;在一个小规模子集上能端到端跑通并产出结果;无需一次性全量运行即可确认工具完整可用。

**Acceptance Scenarios**:

1. **Given** 评测工具随引擎搬入 engram, **When** 在 engram 内构建它, **Then** 构建成功。
2. **Given** 一个小规模评测子集与一个本地(可离线)的模型/向量端点, **When** 运行评测工具, **Then** 端到端完成并产出可比口径的结果。
3. **Given** 一次完整数据集的评测运行, **When** 抽离后运行一次作为完整性抽查(sanity check), **Then** 结果处于合理范围;此完整运行为一次性抽查,不作为逐分合并门禁。

---

### Edge Cases

- 当某个依赖包在宿主里同时服务记忆与非记忆功能(如存储层混有会话/权限数据结构、提示词库混有非记忆模板)时:抽离后 engram 只应包含记忆相关部分;归属存疑的部分**宁可纳入也不遗漏**记忆所需,并在评审中记录归属判断,以免破坏行为。
- 当抽离后引擎在**无外网、无宿主源码**的环境运行时:核心功能(存储、检索、抽取、curation)必须完整可用,不得隐含对任何托管服务或宿主代码的必需依赖。
- 当保真对拍用例中,抽离前基线本身依赖随机性(如答案生成)时:对拍**只锚定确定性的检索层输出**,不把非确定性的下游环节纳入逐条比对,以免用噪声误判回退。
- 当宿主仓库(workhorse-agent)在本次抽离期间发生变化时:本特性**不修改宿主仓库**,以抽离动作发起时的引擎状态为基线;宿主改为消费 engram 是后续独立特性。

## Requirements *(mandatory)*

### Functional Requirements

- **FR-001**: 记忆引擎(存储、混合检索、抽取、curation、向量/嵌入客户端、评测工具)MUST 作为一个自洽的独立库存在于 engram 仓库,能独立于宿主源码构建与运行。
- **FR-002**: 引擎 MUST NOT 依赖任何宿主(会话管理、权限系统、工具生态)代码;当前唯一的宿主工具耦合(查询解析辅助能力)MUST 内化进引擎自身,且行为与抽离前逐字节一致。
- **FR-003**: 抽离前引擎自带的全部单元测试 MUST 随引擎搬入 engram 并全部通过;测试语义 MUST 与抽离前保持一致(仅允许调整引用路径)。
- **FR-004**: 通用基础设施依赖(嵌入客户端、标识生成、LLM 提供方抽象及其具体实现)MUST 完整随引擎搬入,使引擎自带其运转所需的一切。
- **FR-005**: 对于在宿主中混杂了记忆与非记忆内容的共享依赖(存储层、提示词库),engram MUST 只纳入记忆相关部分;非记忆内容(会话/权限结构、非记忆模板)MUST 留在宿主、不进入 engram。
- **FR-006**: engram MUST 拥有一份**只含记忆相关数据结构**的独立存储层,不携带宿主的会话/权限数据结构。
- **FR-007**: 系统 MUST 提供一个确定性的检索对拍验证:在固定语料、固定查询集、固定或打桩向量下,抽离后引擎返回的检索结果(排序后的记忆条目标识序列)MUST 与抽离前基线逐条一致。
- **FR-008**: 该确定性检索对拍 MUST 纳入持续集成,在每次改动时自动重跑。
- **FR-009**: 三路检索信号中任一缺失或失败时,引擎 MUST 优雅降级并仍返回结果,降级行为 MUST 与抽离前一致。
- **FR-010**: 评测/回归工具 MUST 随引擎搬入 engram,能在 engram 内构建,并能在小规模子集上端到端跑通;完整数据集运行作为一次性完整性抽查,MUST NOT 作为逐分合并门禁。
- **FR-011**: 引擎的核心功能路径 MUST 可在无外网、无宿主源码的环境完整运行;任何托管服务 MUST NOT 成为核心路径的必需项。
- **FR-012**: 本特性 MUST NOT 修改宿主仓库(workhorse-agent);宿主改为消费 engram 属后续独立特性。
- **FR-013**: 引擎对外可用的能力面 MUST 与抽离前在语义上等价(公开能力的重新设计——命名空间隔离、错误语义规整、对外兼容 API——不属于本特性,留待后续)。

### Key Entities *(include if feature involves data)*

- **记忆条目 (Memory Entry)**:引擎存储与检索的基本单位,携带其内容、来源、以及支撑分层存储与生物记忆式排序的元数据(如固定标记、留存度、命中次数、事件时间等)。抽离后其数据结构 MUST 保持不变。
- **记忆存储 (Memory Store)**:承载记忆条目及其索引的独立本地存储,抽离后**只含记忆相关结构**,不含宿主会话/权限结构。
- **检索结果 (Retrieval Result)**:对给定查询返回的、按融合排序的记忆条目序列;它是保真对拍逐条比对的对象。
- **评测运行 (Evaluation Run)**:一次对长期对话记忆基准的评测执行,产出可比口径的结果,既用于回归也用于论文实验。

## Success Criteria *(mandatory)*

### Measurable Outcomes

- **SC-001**: 在无外网、无宿主源码的纯净环境中,记忆引擎与评测工具一次性构建成功(缺失依赖数为 0)。
- **SC-002**: 抽离后引擎在 engram 内的既有测试套件通过率为 100%,且相较抽离前无测试丢失。
- **SC-003**: 确定性检索对拍中,抽离后引擎返回的检索结果与抽离前基线的逐条一致率为 100%。
- **SC-004**: 抽离后的引擎代码对宿主(会话/权限/工具生态)代码的引用数为 0。
- **SC-005**: engram 的记忆存储层中,非记忆(会话/权限)数据结构的数量为 0。
- **SC-006**: 评测工具能在小规模子集上端到端跑通并产出结果(可运行性达成)。
- **SC-007**: 完整数据集的一次性完整性抽查结果处于合理范围,未见明显异常回退。

## Assumptions

- 抽离的源头是 workhorse-agent 当前的记忆子系统,该子系统已在宿主内达到生产可用并经长期对话记忆基准多轮消融调优;本特性以其当前状态为保真基线。
- 引擎的核心检索层在给定语料、查询与向量表示下是确定性的,因此可用逐条对拍作为保真证明;下游的答案生成等非确定性环节不纳入逐条比对。
- 评测所需的完整数据集与模型/向量端点由运行者在需要时自行提供;本地(可离线)端点即可满足小子集跑通与完整性抽查。
- 存储层与提示词库在宿主中确实混杂了非记忆内容,抽离需要人工判断记忆相关子集的归属;归属存疑者从宽纳入记忆所需部分。
- 本特性范围内不引入新的对外能力设计;命名空间隔离、错误语义、对外兼容 API 等均为后续独立特性。
- 本特性不修改宿主仓库;宿主继续保有其现有记忆实现,直至后续特性将其改为消费 engram。
