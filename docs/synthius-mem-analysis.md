## 1. 核心论点:检索目标 > 检索方法

所有现有系统(Mem0、TiMem、MemOS、A-MEM、LangMem,以及我们)本质在做同一件事:
**把对话切片,提问时检索"说过什么"**。Synthius-Mem 检索的是**"关于这个人已知
什么"**——把记忆当作一个持续维护的、结构化的人物模型,而不是可搜索的对话日志。

这个目标差异决定了整个架构:提取阶段一次性把认知工作(解析、消歧、时序推理、
相关性过滤)做完,沉淀成高信噪比的结构化事实;检索时拿到的已经是"已知事实",
不是"相似片段"。

## 2. 架构

### 2.1 六大认知域(神经科学启发的功能特化)

记忆不是一个大仓库,而是六个各有独立 schema / 提取模块 / 合并逻辑 / 检索工具
的子系统(**不是同一个向量库贴不同标签,是完全不同的数据结构**):

| 域 | 神经科学对应 | 存什么 | Schema 特点 |
|----|--------------|--------|-------------|
| Biography 生平 | 语义自我认知 | 19 分类原子事实(教育/职业/家庭/健康/居住地…) | 时间戳 + 置信度 |
| Experiences 经历 | 情景记忆 Episodic | 层次化事件树,含情感强度、社会背景 | 时间锚定 + 情感极性 |
| Preferences 偏好 | 评价性记忆 | 喜欢/不喜欢 | 极性、强度、时效状态、原始措辞 |
| Social Circle 社交圈 | 社会认知 | 每个人物模型 | 亲密度、信任度、关系类型 |
| Work 工作 | 专业记忆 | 项目/技能/工具/成果 | 项目结构化 |
| Psychometrics 心理测量 | 元认知/自我模型 | 9 套心理学框架标准化得分 | 数值 + 置信度 + 证据引用 |

Psychometrics 的 9 套框架:Big Five、Schwartz 价值观、PANAS、VIA 品格优势、
认知能力、IRI 共情、道德基础、政治指南针、Kohlberg 道德发展。

**关键:没有可训练参数**。提取 schema = 固定 prompt,合并 = 确定性规则,
检索工具 = 基于规则的字段匹配。是**产品 taxonomy 驱动,不是在 LoCoMo 上训出来的**。

### 2.2 两阶段流程

```
提取阶段(一次性)                        检索阶段(每次提问)
  对话历史                                 用户提问
    ↓ 解析(WhatsApp/TG/PDF/邮件)          ↓
    ↓ 带重叠分块                    规划 LLM 决定查哪个域
    ↓ 并行 LLM 提取到 6 个域               ↓
    ↓ 域内合并去重(Consolidation)   CategoryRAG:域内结构化字段匹配
    ↓ 叙事化总结(传记 + 时间线)          (非向量相似度,均延迟 21.79ms)
    → 冷存储                               ↓
                                    回答 LLM 生成回复
```

- **Consolidation**:去重、合并冲突、层级重构,**域内进行**(教育类事实不会和
  健康类混在一起合并)。
- **CategoryRAG**:规划 LLM 判断查哪个域 → 在结构化 JSON 存储里做**字段级匹配**
  → 搜索空间小,21.79ms 比向量检索还快。

### 2.3 为什么防幻觉(自报 99.6%,442 对抗题错 2 道)

**这是与所有竞品最本质的区别,也是对 engram 最有价值的一点。**

- 检索式系统:问"Melanie 的妹妹叫什么"——她从没提过有妹妹。向量检索**仍会返回
  最相似的 top-K 片段**,把"识别前提不成立"的重担压给回答 LLM,而 LLM 在
  prompt 压力下做这件事不可靠。
- Synthius-Mem:结构化知识库**只存真实出现过的事实**。检索工具查不到匹配 →
  **空结果本身就是机器可读的强信号"前提不成立"** → 直接拒答。

### 2.4 为什么号称比全量上下文强(自报 94.4% vs 85.5%)

全量上下文信息完整,但强制回答 LLM 每次查询自己做全部认知工作,上下文越长
"lost in the middle"越掉精度。Synthius 在提取阶段把认知工作做完一次:
信噪比更高(噪声提取时过滤)、按域组织(问社交圈只查社交圈)、空结果=前提不成立、
时间元数据预解析成结构化字段。**Token 效率**:500 条消息时全量约 26,200 tok/查询,
Synthius 约 5,200——约 5× 节省。

## 3. 与 engram 现状的对照

| 维度 | Synthius-Mem | engram 现状 |
|------|--------------|-------------|
| 检索目标 | "关于这个人已知什么"(人物模型) | "说过什么"(verbatim chunks ∪ 抽取事实) |
| 组织方式 | 6 个独立域 × 独立 schema/检索通道 | 单一混合库,entity 平表 + FTS + 向量 |
| 检索方法 | 规划 LLM 路由 → 域内字段匹配 | **确定性正则** query plan(`queryplan.go`/`enumintent.go`,显式"不携带模型派生状态")+ 三路 RRF |
| 抽取成本 | 重:6 域并行 LLM 提取 + consolidation + 叙事化 | 轻:ADD-only 单通道抽取 |
| 冲突处理 | 域内 consolidation(去重/合并/层级重构) | curation LLM judge(keep/evict/merge)+ 近重复 dedup |
| 拒答 | 空结果=前提不成立=拒答(结构性) | 两级 IDK 重试(实测**害对抗题**:122 错中 121 自信编造) |
| 离线性 | CategoryRAG 字段匹配天然无需向量 | 已具备:无 embedding 时 FTS 优雅降级 |

**两个已被 engram 自己的实测印证的点**:
- Synthius 把 Preferences 做成**独立域 + 独立检索通道**——恰好解释了我们
  v23"把观点条目灌进同一个池子"是负收益(single-hop −25 题)的原因:失败在
  "同池",不在"偏好"本身(见 `memory-strategy.md` 附二短板 3)。
- 规划 LLM 路由 = 查询理解层,与 MemOS MemReader、我们已识别的最大结构性
  短板同向——第三方独立收敛,进一步印证"查询侧结构化"是正确的层。

## 4. 可行性评估(逐机制,ROI 排序)

评估轴:① 在 LoCoMo 上是否真涨分(而非产品叙事)② 是否契合 Go 单二进制 +
单文件 SQLite + 离线 + **引擎/适配器分离**约束 ③ 工程量与宪法 IV 评测门的张力。

### P0 — 空结果=拒答信号(最高 ROI,最便宜,直击真短板)

- **抄什么**:不是六域架构,而是**"检索空/弱 → 前提不成立 → 拒答"**这一条
  单一机制。直接打我们**对抗题 27% 编造率**(文档已列为下一个真实杠杆)。
- **落地**:检索置信度(RRF 融合分 + 命中数 + top-1 与阈值差)低于阈值时,
  答题契约走拒答分支;与已规划的 **Abstain-R1 1:4 ICL 拒答+澄清**(`memory-strategy.md`
  附二短板 4)合流——Synthius 提供了"为什么该拒"的结构性依据,Abstain-R1
  提供了"怎么拒得校准"的实证配方。**两者是互补,不是二选一。**
- **约束契合**:纯适配器/答题层改动,不动引擎算法;可用 `--adversarial` 口径量化。
- **风险**:engram 是**融合检索**,不像 Synthius 有"域内精确匹配→真空结果"的
  干净信号;我们的向量路总会返回 top-K,"空"要靠阈值判定,需消融定标,
  且过严会误拒可答题(Abstain-R1 的不对称惩罚正是为此)。

### P1 — 查询侧域/意图路由(与 MemReader、时间窗过滤合流)

- **抄什么**:规划 LLM 把查询解析成结构化意图(主题/实体/时间范围/目标域),
  在检索侧结构化过滤后再融合——而非把 `[event:]` 渲染给答题模型自己看。
- **落地**:`queryplan.go`/`enumintent.go` 已是**确定性**规划骨架,可在其上**加一层
  可选的 LLM 意图解析**(离线时降级回正则,符合优雅降级纪律);与文档已定的
  P0"检索侧时间窗过滤"(Chronos/MemOS)是同一层的两个动作,应合并立项。
- **约束契合**:触碰引擎检索路径 → **contract-first + 宪法 IV 评测门**,须先冻结
  contract、跑 LoCoMo 不回归再并。规划 LLM 引入网络/延迟,必须 default-off、
  离线可降级。

### P2 — 域内合并 / consolidation 扩展(补 ADD-only 的结构性洞)

- **抄什么**:consolidation 的"域内、冲突消解、层级重构"思路,与我们已规划的
  **冲突消解四分类**(FadeMem,`memory-strategy.md` 附二短板 3)同构。Synthius
  的增量是"按域隔离合并"——教育事实不与健康事实混并,降低错误合并。
- **落地**:curation 的 `memory_merge` 加"类别/域一致性"约束(用现有 entity/
  category 元数据分桶后再判 merge),避免跨主题误合。
- **约束**:引擎 curation 改动,评测门 + 单独 commit(算法与配置分离)。

### 不建议照抄 — 六大认知域全量架构 / Psychometrics

- **六域全量重构**:是**产品 taxonomy 工程**,不是 LoCoMo 涨分点。它要求 6 套
  schema + 6 个提取模块 + 6 套合并 + 规划路由,是数月级投入,且**同时改动引擎的
  抽取/存储/检索三层**,与"引擎不可从适配器侧动、改动须 contract-first + 评测门"
  的宪法直接冲突。收益(结构化/高信噪比)我们可用"P0 拒答 + P1 路由 + P2 域内
  合并"增量逼近,不必推倒重来。
- **Psychometrics(9 套心理框架)**:是 Synthius.ai 的**人物画像商业差异化**
  (persona 产品),对 LoCoMo QA 与 engram"带记忆的本地 Agent 基座"定位
  基本无关,且引入大量主观打分/证据引用负担。**明确不做。**
- **叙事化总结**:长传记叙事对检索精度贡献存疑(反而可能引入"全量上下文"的
  lost-in-the-middle),与 engram 的高信噪比目标相悖。至多作为可选导出物,
  不进检索路径。

## 5. "六大认知域能否扩大范围"—— alphaxiv 文献核查(2026-07)

> 问题:六域是否该扩到更多域,否则无法超过别人?核查了 alphaxiv 上三篇代表作
> (一篇权威综述 + 两篇 2026 新架构),**结论与直觉相反**。

### 5.1 关键发现:六域不是神经科学 taxonomy,是产品内容 taxonomy

学术界公认的记忆分类走的是**正交且更本质的轴,没有一家靠"堆更多内容域"取胜**:

| 论文 | 划分轴 | 与"六域"的关系 |
|------|--------|----------------|
| **AI Meets Brain**(2512.23343,哈工大/NUS/复旦/北大,57 页 ACM 综述) | **nature**(情景 episodic vs 语义 semantic)× **scope**(会话内 inside-trail vs 跨会话 cross-trail)× 短/长期 | 六域是"内容切法",与神经科学两轴**正交**;经典 taxonomy 从不按 biography/work/social 切,而按"信息的性质 + 适用范围" |
| **Memory Beyond Recall / DCPM**(2606.09483,腾讯) | **垂直认知能力层**:原始输入→原子事实→身份 + 信念修订链(supersedes)→域内 schema→**跨域 core schema** | **明确反对**按内容域/存储层组织,主张"往上抽象"才是增量 |
| **Human-Inspired**(2605.08538,微软) | 六大**认知机制**(consolidation / forgetting / maturation / reconsolidation / 知识图 / 混合检索)——是**过程**不是域 | "六"是巧合,指生命周期机制,不是内容域 |

### 5.2 决定性证据:"多切几个域"是错误方向

**DCPM 的头号增益来自"跨域 collision"**——把语义上很远、行为上相似的 schema
跨域连起来(论文原例:"养生" schema 与"园艺/教学" schema,`cos(语义)=0.18` 但
`cos(行为)=0.81`,抽象出"有机培育心智模型"这一跨域 core schema)。这层只占
**1.3% 节点,却扛了消融里 ≈4.1 分**(全系统提升的大头;去掉 supersedes 再掉 2.9 分)。

> **Synthius 严格的六域隔离,恰恰从结构上锁死了 DCPM 拿分最多的机制。** 域切得
> 越干净、越不互通,越拿不到跨域抽象这个真正的杠杆。所以**"扩大范围"若理解为
> "加更多域",不但超不过别人,方向就错了;正确的"扩大"是①往上加认知层
> (事实→信念修订链→schema→跨域抽象)②打通跨域连接。**

### 5.3 诚实前提(沿用文档评测纪律)

- 三篇均跑 **LongMemEval / PersonaMem-v2,不是 LoCoMo,绝对分不可比**,只取
  机制与**论文内部消融相对量**。
- **微软那篇诚实承认在 LongMemEval 上只是"打平不涨分"**(该数据集无冗余/矛盾
  可利用),增益只在运营流(VSCode,+21.8pp 保留精度)上出现——**认知层只在
  对的 regime 才回本**;LoCoMo/单用户持久库是否吃这套,须自证。
- DCPM 用 deepseek-v4-flash / kimi-2.5 + bge-m3 + dense⊕BM25(α=0.7),**技术栈
  与 engram 高度同构**,其读路径**全程无 LLM**(向量命中 + 指针/图遍历),这点
  与 engram 离线约束天然契合。

### 5.4 落到 engram:真正该"扩"的三件事(都往上/往跨域,不往宽)

按 ROI,且都与本文档已定方向或 `memory-strategy.md` 既有 backlog 收敛:

1. **信念修订链(supersedes,不删除)** —— 最便宜、读路径无 LLM、直击 temporal/
   knowledge-update。把 curation 现有的 keep/evict/merge **扩成"新压旧但保留链"**:
   `SUPERSEDES/SUPERSEDEDBY` 双向指针,一次命中 O(k) 遍历取回演化全弧。这是
   ADD-only 抽取当前**缺失的结构性层**,且与 §4 P2 的冲突消解四分类是同一件事。
2. **跨域/跨主题抽象层** —— 正是 `memory-strategy.md` 附二 P0 的实体图 + 质心游走
   的上层目标:DCPM 证明"跨域连接"是多跳与隐式推理的最大杠杆,直击我们
   **multi-hop 卡 66%**。方向是"把域打通",与 Synthius"把域隔离"相反。
3. **nature 轴对齐(近乎免费)** —— engram 的 verbatim chunks ≈ episodic、
   抽取事实 ≈ semantic,已隐含 nature 轴;比再造六个内容域**更本质、成本更低**。

**不该做的"扩"**:把六域扩成 7/10/20 个内容域——taxonomy 工程 treadmill,
边际递减,且违背文献共识与宪法(§4 已论证)。

## 6. 对论文线(`memory-strategy.md` 决策二)的弹药

- Synthius 自报 94.4% / 99.6% 跑在**私有 judge + 自定 taxonomy** 上,是
  **"跨论文分数因协议差异不可比"**论点的又一个一手案例(且闭源不可复现,
  比 MemOS 更甚)。
- "空结果=拒答"是**对抗题记分口径**如何主导"鲁棒性"数字的绝佳对照:
  99.6% 高度依赖数据集里对抗题的比例与判分方式——正是我们要审计的协议差异。
- 六域 vs 单库,与 MemOS 图化、我们 verbatim∪fact union store,共同构成
  **"记忆组织结构如何影响可比性"**的横向对照组。

## 7. 结论与建议动作

1. **立刻可做(适配器层,无引擎风险)**:把 §4 P0"检索空/弱→拒答"与已规划的
   Abstain-R1 拒答契约合并为一个答题层实验,用 `--adversarial` 口径 A/B,
   目标先把 27% 编造率打下来——这是 Synthius 洞察里唯一**便宜且直击真短板**的部分。
2. **进研究 backlog(引擎层,须走 spec-kit + 评测门)**:①**信念修订链
   supersedes**(§5.4-1,读路径无 LLM,与冲突消解四分类合并);②**跨域抽象层**
   (§5.4-2,即 `memory-strategy.md` 附二 P0 实体图的上层目标,直击 multi-hop);
   ③ P1 查询侧 LLM 意图/域路由,与检索侧时间窗过滤(Chronos/MemOS)合并立项。
3. **明确不做**:六域全量重构、**把六域扩成更多内容域**、Psychometrics、
   叙事化进检索路径。
4. **纪律**:凡触碰引擎抽取/检索/curation 的动作,先 contract-first,再跑
   LoCoMo 确认不回归(宪法 IV);算法与评测配置分开 commit。

**一句话**:想超过别人,**方向不是"把六大认知域扩成更多域"**(文献共识 + DCPM
一手证据都表明这是错误方向,且 Synthius 的域隔离恰好锁死了 DCPM 拿分最多的
跨域抽象);正确的"扩大"是**往上加认知层(事实→信念修订链→schema→跨域抽象)+
打通跨域连接**——这与 engram 已从 MemOS/生物启发收敛出的实体图 P0、以及
本文档的拒答校准 P0 完全一致。Synthius 最值钱的仍是"结构化库空结果=强信号"
这一条拒答洞察,其余照既定 P0/P1 推进即可。
